“Review and simplify the chat interface for neurodiverse users (esp. ADHD). Minimize simultaneous notifications, limit choice overload, and reduce clutter.”

“Prioritize message content and user input area. Hide secondary actions (e.g., ‘Doom Scroll Rescue,’ quick chips, timers) behind expandable menus.”

“Implement a ‘focus mode’ button that temporarily hides non-essential UI (chips, timers, connection status cards) to reduce distractions.”

“Use gentle, non-intrusive reminders for inactivity—e.g., fade-in message, not popups. Avoid switching visual states abruptly.”

“Review color and animation usage; avoid rapid color changes and animated pulses unless user-initiated.”

User Rule Prompts
“If the user is inactive, display a single-line prompt above the chat input (e.g., ‘Ready when you are’) and hide timers or elaborate tooltips.”

“For quick actions (chips), only show the most relevant one, with an option for ‘More…’ to keep the interface simple.”

“Mute background status (connected/disconnected) unless it affects message delivery. Provide feedback only if action is required.”

“When in listening (voice) mode, focus only on microphone and transcript. Temporarily hide unrelated buttons or timers.”

Concrete UI/UX Changes
Visual Hierarchy:
Place primary chat interaction elements (input, messages) at center stage; move everything else below, minimized or collapsible.

Reduce Choice Overload:
If >2 chips/options visible, group as a menu instead of inline.

Sensory Simplicity:
Avoid simultaneous animations, blinking, or color shifts.

Guided Focus:
Allow users to toggle focus mode or minimal mode.
